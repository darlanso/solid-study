<h1> O princípio Aberto/Fechado (OCP)</h1>
<p> Um sistema deve ser aberto para a extensão, mas fechado para a modificação.</p>
<p>Isso significa que devemos poder criar novas funcionalidades e estender o sistema sem precisar modificar muitas classes já existentes</p>
<p><strong>Uma classe que tende a crescer "para sempre" é uma forte candidata a sofrer alguma espécie de refatoração.</strong></p>
